Lesson 3 :: HEADER & FOOTER

HEADER(頁首)、FOOTER(頁尾)，寫一個網頁最基本必然具備的東西，這課會解構網頁的Template(模板)的重要性。

  1. HEADER
  2. CONTENT
  3. FOOTER

Template的重要性，道理跟建立一個獨立CSS文件一樣，為了減少重覆性動作而大大降低錯誤出現的機率。

先重新定制網頁結構

  1. /web/ - Project資料夾
  2. ├ ┬ /templates/ - 放置所有網頁文件的資料夾
  3. │ └┬  /common/ - 放置常用網頁文件的資料夾
  4. │  ├─  header.php
  5. │  └─  footer.php
  6. ├ /images/ - 放置圖片的資料夾
  7. ├ /css/ - 放置CSS的資料夾
  8. ├ /js/ - 放置JS的資料夾
  9. └ index.php

HEADER

  1. <!DOCTYPE html>
  2. <html>
  3.   <head>
  4.     <meta charset="utf-8" />
  5.     <title>Web Title</title>
  6.     <link rel="stylesheet" type="text/css" href="style.css" />
  7.   </head>
  8.   <body>
  9.     <header>
  10.         Header Content
  11.     </header>

先獨立開啟一個叫header.php的文件，這部份只會將HTML至BODY以及HEADER的TAG存放在HEADER.PHP內。

CONTENT

  1.     Body Content

每頁的內容側存放在這裡，而接著下來便是建立FOOTER.PHP，後面就是寫PHP格式的文件導入編碼。

FOOTER

  1.     <footer>
  2.       Footer Content
  3.     </footer>
  4.   </body>
  5. </html>

再獨立開啟一個叫footer.php的文件，這部份只會將FOOTER至HTML的TAG存放在FOOTER.PHP內。

學一點PHP

為何不是HTML?

因為單純的HTML是不能以Template導入寫網頁的，如果夾雜了其他language就可以輕鬆寫網頁。所以第一次教要安裝XAMPP就是為了這個目的。

以下是PHP的基本寫法：

  1. <?php {code} ?>

PHP導入網頁的方法：

  1. <?php include ('{path/file}'); ?>
  2. <?php include_once ('{path/file}'); ?>
  3. <?php require ('{path/file}'); ?>
  4. <?php require_once ('{path/file}'); ?>

講乜鳩?

include()：這個函式容許你在腳本中把同一個檔案插入多次，若果檔案不存在，它會發出系統警告並繼續執行腳本。
include_once()：它跟 include() 的功能相似，正如它的名字所示，在腳本的執行期間，有關檔案只會被插入一次。
require()：跟 include() 差不多，它也是用來在腳本中插入其他檔案，但若果檔案不存在，它會發出系統警告，這個警告會引致致命錯誤令腳本中止執行。

{path/file} 在CSS導入式的部份已經教過，同一樣的道理，就是應用左在同的位置。
